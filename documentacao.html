1. Introdução

1.1 Objetivo

Este trabalho implementa um compilador completo para uma linguagem de programação estruturada didática, com tradução para código C. O compilador abrange três fases clássicas: análise léxica, análise sintática e geração de código, todas implementadas em Python 3 sem o uso de bibliotecas de expressões regulares ou geradores automáticos (como PLY, ANTLR ou similares) — todo o reconhecimento é feito manualmente, caractere a caractere.

O ponto de entrada é main.py, que coordena as três fases em sequência e interrompe a compilação caso qualquer fase reporte erros.

2. A Linguagem Fonte

A linguagem fonte é uma linguagem imperativa estruturada, inspirada em Pascal e C, com palavras-chave em português. Programas têm estrutura de bloco explícita delimitada por programa / fimprog, e cada comando termina obrigatoriamente com ponto final (.).

3. Gramática Formal

3.1 Gramática livre de contexto

A gramática abaixo descreve formalmente a linguagem. Ela foi projetada sem recursividade à esquerda nas produções de expressão, viabilizando o parser descendente recursivo.

prog      → 'programa' bloco 'fimprog' '.'

bloco     → cmd bloco
           | ε

cmd       → declara
           | cmdAtrib
           | cmdSe
           | cmdEnquanto
           | cmdFaca
           | cmdPara
           | cmdLeia
           | cmdEscreva

declara   → tipo ID '.'

tipo      → 'inteiro' | 'decimal' | 'texto'

cmdAtrib  → ID ':=' expr '.'

cmdSe     → 'se' '(' exprRel ')' '{' bloco '}' senaoOpt

senaoOpt  → 'senao' '{' bloco '}'
           | ε

cmdEnquanto → 'enquanto' '(' exprRel ')' '{' bloco '}'

cmdFaca   → 'faca' '{' bloco '}' 'enquanto' '(' exprRel ')' '.'

cmdPara   → 'para' '(' cmdAtrib* ';' exprRel ';' cmdAtrib* ')' '{' bloco '}'
            (* sem ponto final na atribuição interna ao para)

cmdLeia   → 'leia' '(' ID ')' '.'

cmdEscreva → 'escreva' '(' conteudo ')' '.'

conteudo  → TEXTO_LIT | expr

exprRel   → expr op_rel expr

op_rel    → '<' | '>' | '<=' | '>=' | '==' | '!='

expr      → termo expr'
expr'     → '+' termo expr'
           | '-' termo expr'
           | ε

termo     → fator termo'
termo'    → '*' fator termo'
           | '/' fator termo'
           | ε

fator     → NUM_INT
           | NUM_DEC
           | VERDADEIRO
           | FALSO
           | ID
           | '(' expr ')'

3.2 Observações sobre a gramática

  • A recursividade à esquerda em expr e termo foi eliminada através da introdução de expr' e termo', tornando a gramática LL(1) compatível com parser descendente recursivo.
  • O look-ahead de 1 token (LL(1)) é suficiente para decidir qual produção aplicar em todos os pontos da gramática.
  • O terminador de comando . é obrigatório em todos os comandos, exceto nas atribuições internas ao para.

4. Análise Léxica

4.1 Funcionamento

O analisador léxico (lexico.py) lê o código-fonte caractere por caractere, mantendo três ponteiros de estado: posição atual (pos), número da linha (linha) e look-ahead de um caractere (_proximo()).

O método principal _proximo_token() descarta espaços e comentários de linha (//) e, com base no caractere corrente, delega para um dos três reconhecedores:

  • _le_palavra() — lê sequências alfanuméricas e verifica se são palavra reservada (via dicionário PALAVRAS_RESERVADAS) ou identificador.
  • _le_numero() — lê inteiros ([0-9]+) e decimais ([0-9]+'.'[0-9]+).
  • _le_operador() — reconhece operadores de um ou dois caracteres usando look-ahead para distinguir < de <=, por exemplo.

4.3 Tratamento de erros léxicos

Erros são registrados na lista self.erros e impressos imediatamente. Após registrar o erro, o léxico tenta recuperar chamando _proximo_token() novamente (recuperação por pânico simples), permitindo que múltiplos erros sejam reportados em uma única execução.

Exemplo de erro léxico:
ERRO LÉXICO na linha 5: Caractere '@' não reconhecido

5. Análise Sintática

5.1 Estratégia: parser descendente recursivo

O analisador sintático (sintatico.py) implementa um parser descendente recursivo LL(1), com uma função Python por regra gramatical. O método _consome(tipo_esperado) verifica o token corrente, avança o ponteiro e retorna o token consumido — ou lança ErroSintatico se o tipo não corresponder.

5.2 Árvore de derivação (AST)

Cada função de regra retorna um objeto No, compondo a árvore de baixo para cima. A classe No possui quatro atributos:

AtributoDescrição
tipoNome da regra ou token (ex.: 'cmdSe', 'ID')
filhosLista de nós filhos (nós internos)
valorTexto do token (somente nas folhas: ID, NUM_INT, etc.)
linhaNúmero da linha no fonte (para mensagens de erro)

Exemplo de árvore para resultado := a + b * 2.:

[cmdAtrib]
  [ID] "resultado"
  [expr]
    [ID] "a"
    [OP] "+"
    [termo]
      [ID] "b"
      [OP] "*"
      [NUM_INT] "2"

5.3 Tratamento de erros sintáticos

Ao encontrar um token inesperado, o método _erro() registra a mensagem, imprime e lança ErroSintatico. O método analisar() captura a exceção e retorna None, sinalizando ao main.py que a compilação deve ser interrompida.

Exemplo de erro sintático:
ERRO SINTÁTICO na linha 8: Esperado 'PONTO', encontrado 'fimprog' (FIMPROG)

6. Geração de Código C

6.1 Estratégia em duas passagens

O gerador (gerador.py) percorre a AST em duas passagens:

  1. 1ª passagem — _coletar_tipos(): percorre toda a árvore coletando declarações de variáveis e montando a tabela de símbolos { nome → tipo_fonte }. Necessário para inferir o formato de printf/scanf correto para cada variável.
  2. 2ª passagem — _gerar_prog(): gera o código C linha a linha, mantendo um contador de indentação (self._indent) incrementado/decrementado nos métodos _abre_bloco() e _fecha_bloco().

6.2 Tabela de símbolos

A tabela é um dicionário simples { str → str } mapeando nome da variável para seu tipo na linguagem fonte ('inteiro', 'decimal' ou 'texto'). Ela é usada para selecionar automaticamente o especificador de formato correto (%d, %lf, %s) e decidir se o operador de endereço & deve ser usado no scanf.

8. Exemplo Completo

Programa fonte (teste.prog)

// Programa de teste — cobre todos os tokens da linguagem
programa

// Declarações de variáveis (3 tipos)
inteiro a.
inteiro b.
inteiro resultado.
decimal media.
texto mensagem.

// Entrada de dados
escreva("Digite o primeiro numero").
leia(a).
escreva("Digite o segundo numero").
leia(b).

// Atribuição e expressão matemática com precedência
resultado := a + b * 2.
media := (a + b) / 2.

// Estrutura if/else
se (a > b) {
    escreva("A e maior").
    resultado := a - b.
} senao {
    escreva("B e maior ou igual").
    resultado := b - a.
}

// if encadeado
se (resultado > 10) {
    escreva("Grande").
} senao {
    se (resultado > 5) {
        escreva("Medio").
    } senao {
        escreva("Pequeno").
    }
}

// Estrutura enquanto (while)
inteiro i.
i := 0.
enquanto (i < resultado) {
    escreva(i).
    i := i + 1.
}

// Estrutura faca/enquanto (do-while)
faca {
    escreva("Pelo menos uma vez").
    i := i - 1.
} enquanto (i > 0).

// Estrutura para (for)
inteiro j.
para (j := 0 ; j < 5 ; j := j + 1) {
    escreva(j).
}

// Operadores relacionais variados
se (a <= b) { escreva("a menor ou igual b"). }
se (a >= b) { escreva("a maior ou igual b"). }
se (a == b) { escreva("iguais"). }
se (a != b) { escreva("diferentes"). }

fimprog.

Código C gerado (teste.c)

#include <stdio.h>
#include <stdlib.h>

int main(void)
{
    int a;
    int b;
    int resultado;
    double media;
    char* mensagem;
    printf("Digite o primeiro numero\n");
    scanf("%d", &a);
    printf("Digite o segundo numero\n");
    scanf("%d", &b);
    resultado = (a + (b * 2));
    media = ((a + b) / 2);
    if (a > b)
    {
        printf("A e maior\n");
        resultado = (a - b);
    }
    else
    {
        printf("B e maior ou igual\n");
        resultado = (b - a);
    }
    if (resultado > 10)
    {
        printf("Grande\n");
    }
    else
    {
        if (resultado > 5)
        {
            printf("Medio\n");
        }
        else
        {
            printf("Pequeno\n");
        }
    }
    int i;
    i = 0;
    while (i < resultado)
    {
        printf("%d\n", i);
        i = (i + 1);
    }
    do
    {
        printf("Pelo menos uma vez\n");
        i = (i - 1);
    }
    while (i > 0);
    int j;
    for (j = 0; j < 5; j = (j + 1))
    {
        printf("%d\n", j);
    }
    if (a <= b)
    {
        printf("a menor ou igual b\n");
    }
    if (a >= b)
    {
        printf("a maior ou igual b\n");
    }
    if (a == b)
    {
        printf("iguais\n");
    }
    if (a != b)
    {
        printf("diferentes\n");
    }

    return 0;
}

9. Como Executar

Pré-requisitos

  • Python 3.10 ou superior
  • Sem dependências externas (bibliotecas padrão apenas)

Execução

# Dentro da pasta compilador/
python main.py teste.prog

A saída no terminal inclui:

  1. Lista completa de tokens reconhecidos pelo léxico
  2. Árvore de derivação (AST) impressa com indentação visual
  3. Código C gerado, impresso na tela
  4. Arquivo .c gravado no mesmo diretório do arquivo fonte

Compilando e executando o código C gerado

gcc teste.c -o teste
./teste

10. Conclusão

O compilador implementado cobre o ciclo completo de tradução de uma linguagem de alto nível para código C, passando pelas fases de análise léxica, análise sintática e geração de código. Todos os recursos da gramática projetada foram implementados e validados com o programa de teste, que exercita os três tipos de dados, todas as estruturas de controle, operadores aritméticos e relacionais, entrada e saída de dados.

Pontos de destaque da implementação:

  • Sem dependências externas — léxico implementado caractere a caractere, sem regex.
  • Parser LL(1) — gramática sem recursividade à esquerda, uma função por regra.
  • Geração em duas passagens — coleta de tipos seguida de geração, garantindo printf/scanf corretos para cada tipo.
  • Relatório de erros com número de linha — em todas as fases do compilador.
